# Support

`ossify` is maintained on a best-effort basis by [`@zay168`](https://github.com/zay168).

If you are using the CLI, the install scripts, or the generated project files and need help, open a GitHub issue with enough context for someone else to reproduce the situation.

## When to open what

- Open a bug report when something is broken or behaves unexpectedly.
- Open a feature request when you want a new capability, rule, or workflow improvement.
- Open a usage issue when you need guidance on commands, config, templates, or output interpretation.
- Do **not** use public issues for vulnerabilities or exploit details. See [SECURITY.md](./SECURITY.md).

## What to include

Please include:

- your `ossify` version
- your OS and shell
- the command you ran
- the target repository shape, if relevant
- the output or screenshot that explains the problem
- what you expected instead

## Scope

Support is best effort for:

- the `ossify` CLI itself
- PowerShell and POSIX install scripts published by the project
- templates and workflow files generated by `init` and `fix`
- the public landing and docs that explain how to use the project

## Joining the project

If you are interested in contributing regularly rather than just opening one-off issues, that is welcome. Start with focused contributions, good issue reports, or documentation improvements, then reach out through GitHub if you want to help shape the project more directly.
