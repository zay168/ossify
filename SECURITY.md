# Security Policy

Please do not report security issues in public issues.

## How to report

Report vulnerabilities privately to the maintainer instead of opening a public issue. If you already have a private contact path with [`@zay168`](https://github.com/zay168), use it. Otherwise, request a private reporting channel without posting exploit details publicly.

## What to include

Please include:

- a clear description of the issue
- affected command, workflow, or generated file
- reproduction steps
- impact and realistic abuse scenario
- versions and environment details
- any mitigation or workaround you already tested

## Scope

This policy covers, on a best-effort basis:

- the `ossify` CLI
- install scripts published by the project
- templates and generated files emitted by `init` and `fix`
- GitHub workflow files and other repo assets published by this repository

If the issue is in an upstream dependency, please consider reporting it upstream as well.

## Response expectations

Security reports are handled on a best-effort basis. Credible reports will be acknowledged as quickly as practical, investigated, and handled through a responsible disclosure process.
